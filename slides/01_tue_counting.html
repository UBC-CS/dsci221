Counting Operations

What counts as a “step”?

We count primitive operations:

  • Arithmetic: +, -, *, /
  • Comparisons: <, >, ==
  • Assignment: x = 5
  • Array/list access: arr[i]
  • Function calls (not counting the work inside)

Key insight: Each takes roughly constant time.

Example: Finding the maximum

Total: 1 + n comparisons + (up to n assignments) + 1

Roughly 2n + 2 operations. We say this is O(n).

Why “roughly”?

We don’t care about:

  • Constants: 2n vs 3n vs 100n — all O(n)
  • Lower-order terms: n² + 5n + 17 — just O(n²)

We care about: How does the count grow as n grows?

The formal definition

Big-O Notation

f(n) is O(g(n)) if there exist constants c > 0 and n₀ ≥ 0 such that:

\[f(n) \leq c \cdot g(n) \quad \text{for all } n \geq n_0\]

In words: f(n) grows no faster than g(n), once n is big enough.

Picture this

For n ≥ 3, we have 2n + 3 ≤ 3n. So f(n) = O(n).

Choose c = 3, n₀ = 3.

Analyzing Loops

Single loops

n iterations × O(1) work = O(n)

Nested loops

n iterations × n iterations × O(1) work = O(n²)

Nested loops (variant)

Iteration 0: 0 prints Iteration 1: 1 print Iteration 2: 2 prints … Iteration n-1: n-1 prints

Total: 0 + 1 + 2 + … + (n-1) = n(n-1)/2 = O(n²)

The triangle sum

Useful formula

\[1 + 2 + 3 + \ldots + n = \frac{n(n+1)}{2}\]

This is O(n²), not O(n).

Whenever you see “for each pair” or “compare all pairs,” expect O(n²).

Sequential code

Total: O(n) + O(n²) + O(n) = O(n²)

The largest term dominates.

Big-O in Practice

Common patterns

Code pattern Big-O
Single loop over n items O(n)
Nested loop (both over n) O(n²)
Loop that halves each time O(log n)
Nested: outer halves, inner is n O(n log n)

The halving pattern

How many times can you halve n before reaching 1?

log₂(n) times. This is O(log n).

Formalizing Big-O

Other notations (preview)

Notation Meaning Analogy
O(g(n)) ≤ g(n) (upper bound) “at most”
Ω(g(n)) ≥ g(n) (lower bound) “at least”
Θ(g(n)) = g(n) (tight bound) “exactly”

We’ll use O most of the time. It’s the “worst-case guarantee.”

Proving Big-O

To prove f(n) = O(g(n)), find constants c and n₀:

Example: Prove 3n² + 5n + 2 = O(n²)

For n ≥ 1: - 3n² + 5n + 2 ≤ 3n² + 5n² + 2n² = 10n²

Choose c = 10, n₀ = 1. ✓

Common Big-O facts

Useful rules

  1. Constants don’t matter: 5n = O(n)
  2. Lower terms don’t matter: n² + n = O(n²)
  3. Logs don’t matter for base: log₂(n) = O(log₁₀(n))
  4. Polynomials: nᵃ = O(nᵇ) if a ≤ b
  5. Exponentials dominate polynomials: n¹⁰⁰ = O(2ⁿ)
